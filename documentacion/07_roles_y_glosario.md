# Roles y Glosario

## 1. Roles del sistema

### Administrador
- Gestiona usuarios y configuracion general
- Administra cursos y estudiantes
- Tiene acceso amplio a modulos de operacion

### Inspector
- Registra asistencia
- Registra atrasos
- Registra y revisa salidas anticipadas
- Consulta informacion por curso o estudiante

### Docente
- Consulta informacion operativa de sus cursos
- Revisa asistencia y eventos relevantes

### Asistente
- Apoya tareas administrativas y consultas operativas

### Apoderado
- Rol futuro para consultas o notificaciones, sujeto a evolucion del producto

## 2. Glosario

### Asistencia
Registro del ingreso diario de un estudiante.

### Atraso
Condicion asignada cuando el ingreso se produce fuera del horario esperado.

### Salida anticipada
Retiro del estudiante antes del termino de la jornada.

### Autorizacion
Validacion interna que respalda una salida anticipada.

### Trazabilidad
Capacidad de conocer quien realizo una accion y cuando.

### Jornada
Periodo diario en el que el estudiante permanece bajo supervision escolar.

### Reglas de negocio
Condiciones que determinan como se debe comportar el sistema frente a una operacion.

### Integridad referencial
Garantia de que las relaciones entre entidades validas se mantengan consistentes.

## 3. Terminologia recomendada

Para mantener consistencia en toda la documentacion, se recomienda usar siempre:

- Estudiante, no alumno, salvo que se cite un documento externo
- Curso, no nivel, salvo que exista una clasificacion especifica
- Salida anticipada, no permiso de salida, para mantener una sola denominacion
- Atraso, no tardanza, para mantener uniformidad
