# Roadmap y Entregables

## 1. Objetivo del roadmap

Definir una ruta de evolucion razonable para convertir la definicion del producto en una plataforma operativa, entregando valor temprano y reduciendo riesgos.

## 2. Fase 0 - Definicion

### Entregables
- Vision del producto
- Alcance inicial
- Epicas e historias de usuario
- Requisitos funcionales y no funcionales
- Modelo de datos conceptual

### Resultado esperado
Existe una base clara para estimacion, priorizacion y desarrollo.

## 3. Fase 1 - Nucleo operativo

### Entregables
- Autenticacion y control de acceso
- Gestion de cursos
- Gestion de estudiantes
- Registro de asistencia
- Deteccion de atrasos

### Resultado esperado
El equipo puede operar el flujo diario principal del establecimiento.

## 4. Fase 2 - Trazabilidad ampliada

### Entregables
- Salidas anticipadas
- Auditoria de autorizaciones
- Consultas por curso, estudiante y fecha
- Vistas de seguimiento operacional

### Resultado esperado
Existe control sobre eventos especiales y mayor trazabilidad historica.

## 5. Fase 3 - Reporteria

### Entregables
- Exportacion a XLSX
- Exportacion a PDF
- Resumenes por periodo
- Indicadores operativos

### Resultado esperado
La informacion se convierte en soporte para supervision y analisis.

## 6. Fase 4 - Automatizacion futura

### Entregables posibles
- Notificaciones a apoderados
- Alertas por atraso recurrente
- Reglas parametrizables por establecimiento
- Integraciones externas

### Resultado esperado
El sistema reduce trabajo manual y aumenta la capacidad de respuesta.

## 7. Criterios de priorizacion

Se recomienda priorizar segun:

- Impacto operativo diario
- Riesgo de no implementar
- Frecuencia de uso
- Esfuerzo de desarrollo
- Dependencias tecnicas

## 8. Entregables de documentacion a mantener

- Backlog priorizado
- Version de requisitos
- Matriz de roles y permisos
- Diccionario de datos
- Diagramas actualizados
- Registro de decisiones tecnicas

## 9. Indicadores de avance

- Historias definidas vs. historias pendientes
- Cobertura de pruebas por modulo
- Porcentaje de funcionalidades operativas
- Tiempo promedio de consulta por flujo clave
- Nivel de adopcion por usuario

## 10. Proxima iteracion recomendada

La siguiente evolucion documental deberia incluir:

- Matriz de roles y permisos detallada
- Flujo de aprobacion de salidas anticipadas
- Catalogo de estados para asistencia y eventos
- Diccionario de API
- Politicas de retencion de datos
